Step-By-Step Setup

Create a read-only AWS role

Create this role once, then paste the role ARN and external ID to run your AWS Savings Review.

1

Enable Cost Explorer

In AWS Billing and Cost Management, turn on Cost Explorer. This powers the spend baseline and other billing checks.

2

Enable Cost Optimization Hub

Turn on Cost Optimization Hub so AWS-native recommendation coverage can populate before the first report.

3

Create a new IAM role in your AWS account

Name it something like aws-savings-review-readonly. Trust only the AWS account below and require the external ID shown here.

Trust Policy

Paste this into the role trust policy

Trusted AWS account {{ settings.operator_trusted_account_id }}

This policy allows the review service to assume the role only when the exact external ID below is provided.

Suggested external ID {{ suggested_external_id }} 
{{ trust_policy }}

Permissions Policy

Attach this read-only permissions policy

Access mode Read-only

This policy grants read-only access to billing, inventory, and optimization signals without making changes in your account. 

{{ permissions_policy }}

What To Paste Back Into The Site

4

Copy the role ARN

Example: arn:aws:iam::123456789012:role/aws-savings-review-readonly

5

Use the same external ID

The external ID in the trust policy must exactly match what you paste into the submission form.

6

Return here and continue

Paste the role ARN and external ID into the form to validate access and start the review.
