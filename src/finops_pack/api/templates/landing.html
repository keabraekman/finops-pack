How It Works

1

Enable the billing prerequisites

Turn on Cost Explorer and Cost Optimization Hub in AWS Billing and Cost Management.

2

Create the read-only role

Use the trusted AWS account ID {{ trusted_account_id }} and a customer-specific external ID.

3

Paste the role ARN and external ID

Paste the role details into the form and confirm the billing prerequisites are enabled.

4

Validate access

The site checks the role and billing setup before the review starts.

5

Add your work email

Your email is used to send the report link and save this run to your report history.

6

Review the report

Open the dashboard in-browser, then use the appendix for supporting evidence and validation detail.

Trust and simplicity

One role, read-only access, no shared credentials.

  • Read-only cross-account IAM role only
  • External ID required for assume-role
  • No AWS passwords, no console login sharing, no long-term access keys
  • Single-account and AWS Organization reviews supported

Who it is for

A quick outside review for teams with meaningful AWS spend.

  • Founders, operators, platform, infrastructure, and finance owners
  • Teams that want a fast review without a long onboarding project
  • Anyone wanting a practical outside perspective on AWS cost

Coverage

Covers active AWS regions by default.

The review scans active commercial AWS regions automatically when role permissions allow it, with a fallback coverage set for local testing.

Spend window

Uses recent billing data.

Spend figures reflect current Cost Explorer data rather than lifetime spend since the account was created.
